# 模式：apply -- 实时申请助手

适用于候选人正在浏览器里填写申请表的场景。读取当前页面、加载已有职位上下文，并针对表单里的每个问题生成定制化回答。

## 前提

- **最佳体验：可见的 Playwright**。候选人能看到浏览器，Claude 也能读取页面内容
- **没有 Playwright**：候选人提供截图，或把表单问题手动贴出来

## Workflow

```
1. DETECT     -> 读取当前 Chrome tab（截图 / URL / 标题）
2. IDENTIFY   -> 提取公司名 + 岗位名
3. SEARCH     -> 在已有 reports/ 中查找上下文
4. LOAD       -> 读取完整 report + Section G（如果存在）
5. COMPARE    -> 当前岗位是否和已评估岗位一致？如果变了要提醒
6. ANALYZE    -> 识别当前页面可见的全部问题
7. GENERATE   -> 针对每个问题生成定制回答
8. PRESENT    -> 用便于 copy-paste 的格式输出
```

## 第 1 步 -- 检测当前岗位

**有 Playwright：** 对当前页面做 snapshot，读取标题、URL 和可见内容。

**没有 Playwright：** 让候选人：

- 分享表单截图（Read tool 可以读图）
- 或直接粘贴表单问题
- 或至少告诉我们公司名 + 岗位名，方便查已有上下文

## 第 2 步 -- 识别并加载上下文

1. 从页面中提取公司名和岗位名
2. 在 `reports/` 中按公司名搜索（大小写不敏感）
3. 如果找到 match，就读取完整 report
4. 如果 report 有 Section G，也一并读取，作为草稿基础
5. 如果没找到，就提醒用户并建议先跑一次快速 auto-pipeline

## 第 3 步 -- 识别岗位是否变化

如果当前页面岗位和已有评估不一致：

- **先提醒候选人**：“当前岗位已从 [X] 变成 [Y]。要不要重新评估，还是沿用现有素材先适配？”
- **如果选择适配**：在不重跑 A-F 的情况下，按新标题重写回答
- **如果选择重评估**：重新执行 A-F，更新 report，重建 Section G
- **必要时更新 tracker**：如果岗位标题确认变了，就把 `applications.md` 中的 title 同步

## 第 4 步 -- 分析表单问题

识别当前可见的全部问题类型：

- 自由文本（例如 why this role、cover letter、why this company）
- 下拉框（例如 how did you hear about us、work authorization）
- Yes / No（例如 relocation、visa、是否接受 hybrid）
- 薪资字段（预期薪资、当前薪资、总包范围）
- 日期 / 地点字段（最快到岗时间、工作城市、是否可驻场 / 出差）
- 上传字段（简历、求职信 PDF）

并把问题分成两类：

- **Section G 已经答过** → 基于现有版本微调
- **全新问题** → 依据 report + `cv.md` 生成新回答

## 第 5 步 -- 生成回答

每个问题都按以下顺序生成：

1. **先用 report 上下文**：优先引用 B 区块 proof points、F 区块 STAR 故事
2. **再用已有 Section G**：如果已有草稿，就用它做基础并精修
3. **保持 “我在选择你们” 的语气**：不是跪求，不是模板腔
4. **必须具体**：引用当前 JD 里的职责、产品、团队或技术栈
5. **必要时带上 portfolio / demo**：如果表单里有 “Additional info” 之类字段，就把最相关的 proof point 放进去

**对中国大陆常见表单字段的处理：**

- **薪资期望**：优先报区间，并说明看重总包结构与职责范围
- **最快到岗时间**：按真实离职周期回答，不要拍脑袋缩短
- **工作地点 / base**：明确能否接受某城市、驻场或出差
- **身份 / 签证 / work authorization**：按真实情况写
- **是否接受混合办公**：按 `profile.yml` 的 location policy 回答

**输出格式：**

```
## [公司] -- [岗位] 的表单回答

基于：Report #NNN | Score: X.X/5 | Archetype: [类型]

---

### 1. [表单原问题]
> [可直接复制的回答]

### 2. [下一题]
> [回答]

...

---

备注：
- [关于岗位变化、信息缺失等观察]
- [候选人最好手动再检查一遍的地方]
```

## 第 6 步 -- 投递后（可选）

如果候选人确认已经提交申请：

1. 把 `applications.md` 状态从 `Evaluated` 更新为 `Applied`
2. 把最终版回答写回 report 的 Section G
3. 建议下一步执行 `/career-ops contact` 做 LinkedIn / 脉脉外联

## 滚动处理

如果表单问题多于一屏：

- 让候选人继续滚动后再发截图
- 或把剩余问题直接贴出来
- 分轮处理，直到整张表单覆盖完整
